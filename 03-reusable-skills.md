# Module 03 — Reusable skills

## Goal

Capture a method that produces dependable results, not just a prompt that sounds good once.

## Practice

Document a skill in four parts:

1. **When to use it** — the trigger and scope.
2. **Inputs** — files, decisions, or commands it needs.
3. **Method** — the ordered steps and quality checks.
4. **Output** — exact sections, artifacts, and stop conditions.

Start with something low-risk: release-note drafting, API change review, or writing a test plan. Keep the skill read-only until the process is trustworthy.

## Check

Run it on two different tasks. If the result is inconsistent, tighten the output format before adding more instructions.

## Next step

Give a narrowly scoped skill to a specialist role.
