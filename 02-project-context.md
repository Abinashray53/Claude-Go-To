# Module 02 — Project context

## Goal

Give an assistant the information a new teammate would need before making a safe change.

## Practice

Add a root-level `PROJECT_CONTEXT.md` with:

- what the project does and who uses it;
- the main directories and their ownership;
- how to run tests and formatting;
- non-negotiable conventions and security boundaries;
- a definition of done for a small pull request.

Link to deeper documents instead of trying to write a novel. Context should be accurate, short, and revised when the project changes.

## Check

Ask someone unfamiliar with the repository to use only this file to locate the test command. If they cannot, improve the file.

## Next step

Use the context file as the foundation for a reusable skill.
